## Summary
<h2>What's changed</h2>
<ul>
<li>Added <code>rate_limits</code> field to statusline scripts for displaying Claude.ai rate limit usage (5-hour and 7-day windows with <code>used_percentage</code> and <code>resets_at</code>)</li>
<li>Added <code>source: 'settings'</code> plugin marketplace source — declare plugin entries inline in settings.json</li>
<li>Added CLI tool usage detection to plugin tips, in addition to file pattern matching</li>
<li>Added <code>effort</code> frontmatter support for skills and slash commands to override the model effort level when invoked</li>
<li>Added <code>--channels</code> (research preview) — allow MCP servers to push messages into your session</li>
<li>Fixed <code>--resume</code> dropping parallel tool results — sessions with parallel tool calls now restore all tool_use/tool_result pairs instead of showing <code>[Tool result missing]</code> placeholders</li>
<li>Fixed voice mode WebSocket failures caused by Cloudflare bot detection on non-browser TLS fingerprints</li>
<li>Fixed 400 errors when using fine-grained tool streaming through API proxies, Bedrock, or Vertex</li>
<li>Fixed <code>/remote-control</code> appearing for gateway and third-party provider deployments where it cannot function</li>
<li>Fixed <code>/sandbox</code> tab switching not responding to Tab or arrow keys</li>
<li>Improved responsiveness of <code>@</code> file autocomplete in large git repositories</li>
<li>Improved <code>/effort</code> to show what auto currently resolves to, matching the status bar indicator</li>
<li>Improved <code>/permissions</code> — Tab and arrow keys now switch tabs from within a list</li>
<li>Improved background tasks panel — left arrow now closes from the list view</li>
<li>Simplified plugin install tips to use a single <code>/plugin install</code> command instead of a two-step flow</li>
<li>Reduced memory usage on startup in large repositories (~80 MB saved on 250k-file repos)</li>
<li>Fixed managed settings (<code>enabledPlugins</code>, <code>permissions.defaultMode</code>, policy-set env vars) not being applied at startup when <code>remote-settings.json</code> was cached from a prior session</li>
</ul>
