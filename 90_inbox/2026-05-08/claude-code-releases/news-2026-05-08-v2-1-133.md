## Summary
<h2>What's changed</h2>
<ul>
<li>Added <code>worktree.baseRef</code> setting (<code>fresh</code> | <code>head</code>) to choose whether <code>--worktree</code>, <code>EnterWorktree</code>, and agent-isolation worktrees branch from <code>origin/&lt;default&gt;</code> or local <code>HEAD</code>. <strong>Note:</strong> the default <code>fresh</code> changes <code>EnterWorktree</code>'s base back to <code>origin/&lt;default&gt;</code> (it has been local <code>HEAD</code> since 2.1.128) — set <code>worktree.baseRef: "head"</code> to keep unpushed commits in new worktrees</li>
<li>Added <code>sandbox.bwrapPath</code> and <code>sandbox.socatPath</code> managed settings (Linux/WSL) to specify custom bubblewrap and socat binary locations</li>
<li>Added <code>parentSettingsBehavior</code> admin-tier key (<code>'first-wins' | 'merge'</code>) to let admins opt SDK <code>managedSettings</code> (parent tier) into the policy merge</li>
<li>Hooks now receive the active effort level via the <code>effort.level</code> JSON input field and the <code>$CLAUDE_EFFORT</code> environment variable, and Bash tool commands can read <code>$CLAUDE_EFFORT</code></li>
<li>Improved focus mode behavior</li>
<li>Improved memory usage by releasing warm-spare background workers under memory pressure</li>
<li>Fixed parallel sessions all dead-ending at 401 after a refresh-token race wiped shared credentials</li>
<li>Fixed <code>Edit</code>/<code>Write</code> allow rules scoped to a drive root (<code>C:\</code>) or POSIX <code>/</code> matching incorrectly and always prompting</li>
<li>Fixed an unhandled rejection (<code>ECOMPROMISED</code>) when a history or session-log file lock is compromised by clock skew or slow disk</li>
<li>Fixed pressing Esc during conversation compaction showing a spurious "Error compacting conversation" notification</li>
<li>Fixed <code>HTTP(S)_PROXY</code> / <code>NO_PROXY</code> / mTLS not being respected for the full MCP OAuth flow including discovery, dynamic client registration, token exchange, and token refresh</li>
<li>Fixed Read/Write/Edit being denied on mapped network drives passed via <code>--add-dir</code> / SDK <code>additionalDirectories</code></li>
<li>Fixed Remote Control stop/interrupt from claude.ai not fully canceling the CLI session the same way local Esc does, causing queued messages to never advance after interrupting a stuck tool or prompt</li>
<li>Fixed <code>/effort</code> in one session unexpectedly changing the effort level of other concurrent sessions, and a related issue where an IDE effort change could be silently dropped</li>
<li>Fixed subagents not discovering project, user, or plugin skills via the Skill tool</li>
<li><code>claude --help</code> now lists <code>--remote-control</code> alongside <code>--remote-control-session-name-prefix</code></li>
<li>[VSCode] Fixed <code>claudeCode.claudeProcessWrapper</code> failing with "Unsupported platform" when the extension build doesn't bundle a Claude binary</li>
</ul>
