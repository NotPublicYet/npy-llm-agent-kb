## Summary
<h2>What's changed</h2>
<ul>
<li>Added new <code>claude agents</code> flags: <code>--add-dir</code>, <code>--settings</code>, <code>--mcp-config</code>, <code>--plugin-dir</code>, <code>--permission-mode</code>, <code>--model</code>, <code>--effort</code>, and <code>--dangerously-skip-permissions</code> to configure dispatched background sessions</li>
<li>Fast mode now uses Opus 4.7 by default (previously Opus 4.6). Set <code>CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1</code> to pin fast mode to Opus 4.6</li>
<li>Plugins with a root-level <code>SKILL.md</code> and no <code>skills/</code> subdirectory are now surfaced as a skill</li>
<li>The <code>/plugin</code> details pane and <code>claude plugin details</code> now show LSP servers a plugin provides</li>
<li><code>/web-setup</code> warns before replacing an existing GitHub App connection</li>
<li>Fixed <code>MCP_TOOL_TIMEOUT</code> not raising the per-request fetch timeout for remote HTTP and SSE MCP servers, which capped tool calls at 60 seconds regardless of the configured value</li>
<li>Fixed background sessions not recognizing pre-existing git worktrees, blocking Edit while EnterWorktree refused to create a duplicate</li>
<li>Fixed background sessions disappearing and daemon reconnect failing after macOS sleep/wake — the daemon now detects clock jumps instead of treating them as elapsed idle time</li>
<li>Fixed daemon not exiting cleanly after the binary is upgraded (e.g. <code>brew upgrade</code>), causing dispatched agents to crash-loop on the deleted path</li>
<li>Fixed background agents crash-looping when the Claude-in-Chrome extension is connected without a shared tab</li>
<li>Fixed clicking links in an attached <code>claude agents</code> session — the background worker's headless browser shim no longer applies while attached</li>
<li>Fixed <code>claude agents</code> "v to open in editor" using the daemon's default editor instead of your shell's <code>$EDITOR</code>/<code>$VISUAL</code></li>
<li>Fixed <code>claude agents</code> deadlocking on Windows with network-drive working directories; Ctrl+C now works during startup</li>
<li>Fixed background-color bleed when attaching to a <code>claude agents</code> session from Apple Terminal or other 256-color-only terminals</li>
<li>Fixed <code>claude --bg --dangerously-skip-permissions</code> not persisting across retire/wake</li>
<li>Fixed session titles being derived from the URL when the first message is a link</li>
<li>Fixed redundant <code>set_model</code> requests from remote clients injecting duplicate <code>/model</code> breadcrumbs into the transcript</li>
<li>Fixed plugins using <code>skills: ["./"]</code> showing a false "path escapes plugin directory" error</li>
<li>Fixed plugin cache cleanup deleting the active plugin version directory when no installation metadata is present</li>
<li>Fixed <code>/plugin</code> browse pane showing "0 installs" for newly published plugins</li>
<li>Fixed plugin advisories not naming every <code>plugin.json</code> key that shadows a default folder</li>
<li>Improved reactive compaction: the first summarize attempt now seeds from the original request's overflow size, avoiding a wasted near-full-context retry</li>
<li>Improved hook configuration error: configuring a prompt- or agent-type hook for <code>SessionStart</code>/<code>Setup</code>/<code>SubagentStart</code> now shows a clear "use a command-type hook instead" error</li>
<li>Removed stale <code>/model claude-sonnet-4-20250514</code> suggestion from Usage Policy refusal messages</li>
</ul>
