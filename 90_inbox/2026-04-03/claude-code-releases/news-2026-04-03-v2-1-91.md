## Summary
<h2>What's changed</h2>
<ul>
<li>Added MCP tool result persistence override via <code>_meta["anthropic/maxResultSizeChars"]</code> annotation (up to 500K), allowing larger results like DB schemas to pass through without truncation</li>
<li>Added <code>disableSkillShellExecution</code> setting to disable inline shell execution in skills, custom slash commands, and plugin commands</li>
<li>Added support for multi-line prompts in <code>claude-cli://open?q=</code> deep links (encoded newlines <code>%0A</code> no longer rejected)</li>
<li>Plugins can now ship executables under <code>bin/</code> and invoke them as bare commands from the Bash tool</li>
<li>Fixed transcript chain breaks on <code>--resume</code> that could lose conversation history when async transcript writes fail silently</li>
<li>Fixed <code>cmd+delete</code> not deleting to start of line on iTerm2, kitty, WezTerm, Ghostty, and Windows Terminal</li>
<li>Fixed plan mode in remote sessions losing track of the plan file after a container restart, which caused permission prompts on plan edits and an empty plan-approval modal</li>
<li>Fixed JSON schema validation for <code>permissions.defaultMode: "auto"</code> in settings.json</li>
<li>Fixed Windows version cleanup not protecting the active version's rollback copy</li>
<li><code>/feedback</code> now explains why it's unavailable instead of disappearing from the slash menu</li>
<li>Improved <code>/claude-api</code> skill guidance for agent design patterns including tool surface decisions, context management, and caching strategy</li>
<li>Improved performance: faster <code>stripAnsi</code> on Bun by routing through <code>Bun.stripANSI</code></li>
<li>Edit tool now uses shorter <code>old_string</code> anchors, reducing output tokens</li>
</ul>
