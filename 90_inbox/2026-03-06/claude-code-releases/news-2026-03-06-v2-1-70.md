## Summary
<h2>What's changed</h2>
<ul>
<li>Fixed API 400 errors when using <code>ANTHROPIC_BASE_URL</code> with a third-party gateway — tool search now correctly detects proxy endpoints and disables <code>tool_reference</code> blocks</li>
<li>Fixed <code>API Error: 400 This model does not support the effort parameter</code> when using custom Bedrock inference profiles or other model identifiers not matching standard Claude naming patterns</li>
<li>Fixed empty model responses immediately after <code>ToolSearch</code> — the server renders tool schemas with system-prompt-style tags at the prompt tail, which could confuse models into stopping early</li>
<li>Fixed prompt-cache bust when an MCP server with <code>instructions</code> connects after the first turn</li>
<li>Fixed Enter inserting a newline instead of submitting when typing over a slow SSH connection</li>
<li>Fixed clipboard corrupting non-ASCII text (CJK, emoji) on Windows/WSL by using PowerShell <code>Set-Clipboard</code></li>
<li>Fixed extra VS Code windows opening at startup on Windows when running from the VS Code integrated terminal</li>
<li>Fixed voice mode failing on Windows native binary with "native audio module could not be loaded"</li>
<li>Fixed push-to-talk not activating on session start when <code>voiceEnabled: true</code> was set in settings</li>
<li>Fixed markdown links containing <code>#NNN</code> references incorrectly pointing to the current repository instead of the linked URL</li>
<li>Fixed repeated "Model updated to Opus 4.6" notification when a project's <code>.claude/settings.json</code> has a legacy Opus model string pinned</li>
<li>Fixed plugins showing as inaccurately installed in <code>/plugin</code></li>
<li>Fixed plugins showing "not found in marketplace" errors on fresh startup by auto-refreshing after marketplace installation</li>
<li>Fixed <code>/security-review</code> command failing with <code>unknown option merge-base</code> on older git versions</li>
<li>Fixed <code>/color</code> command having no way to reset back to the default color — <code>/color default</code>, <code>/color gray</code>, <code>/color reset</code>, and <code>/color none</code> now restore the default</li>
<li>Fixed a performance regression in the <code>AskUserQuestion</code> preview dialog that re-ran markdown rendering on every keystroke in the notes input</li>
<li>Fixed feature flags read during early startup never refreshing their disk cache, causing stale values to persist across sessions</li>
<li>Fixed <code>permissions.defaultMode</code> settings values other than <code>acceptEdits</code> or <code>plan</code> being applied in Claude Code Remote environments — they are now ignored</li>
<li>Fixed skill listing being re-injected on every <code>--resume</code> (~600 tokens saved per resume)</li>
<li>Fixed teleport marker not rendering in VS Code teleported sessions</li>
<li>Improved error message when microphone captures silence to distinguish from "no speech detected"</li>
<li>Improved compaction to preserve images in the summarizer request, allowing prompt cache reuse for faster and cheaper compaction</li>
<li>Improved <code>/rename</code> to work while Claude is processing, instead of being silently queued</li>
<li>Reduced prompt input re-renders during turns by ~74%</li>
<li>Reduced startup memory by ~426KB for users without custom CA certificates</li>
<li>Reduced Remote Control <code>/poll</code> rate to once per 10 minutes while connected (was 1–2s), cutting server load ~300×. Reconnection is unaffected — transport loss immediately wakes fast polling.</li>
<li>[VSCode] Added spark icon in VS Code activity bar that lists all Claude Code sessions, with sessions opening as full editors</li>
<li>[VSCode] Added full markdown document view for plans in VS Code, with support for adding comments to provide feedback</li>
<li>[VSCode] Added native MCP server management dialog — use <code>/mcp</code> in the chat panel to enable/disable servers, reconnect, and manage OAuth authentication without switching to the terminal</li>
</ul>
