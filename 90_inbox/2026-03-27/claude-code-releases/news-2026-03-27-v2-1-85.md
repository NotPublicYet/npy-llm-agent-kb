## Summary
<h2>What's changed</h2>
<ul>
<li>Added <code>CLAUDE_CODE_MCP_SERVER_NAME</code> and <code>CLAUDE_CODE_MCP_SERVER_URL</code> environment variables to MCP <code>headersHelper</code> scripts, allowing one helper to serve multiple servers</li>
<li>Added conditional <code>if</code> field for hooks using permission rule syntax (e.g., <code>Bash(git *)</code>) to filter when they run, reducing process spawning overhead</li>
<li>Added timestamp markers in transcripts when scheduled tasks (<code>/loop</code>, <code>CronCreate</code>) fire</li>
<li>Added trailing space after <code>[Image #N]</code> placeholder when pasting images</li>
<li>Deep link queries (<code>claude-cli://open?q=…</code>) now support up to 5,000 characters, with a "scroll to review" warning for long pre-filled prompts</li>
<li>MCP OAuth now follows RFC 9728 Protected Resource Metadata discovery to find the authorization server</li>
<li>Plugins blocked by organization policy (<code>managed-settings.json</code>) can no longer be installed or enabled, and are hidden from marketplace views</li>
<li>PreToolUse hooks can now satisfy <code>AskUserQuestion</code> by returning <code>updatedInput</code> alongside <code>permissionDecision: "allow"</code>, enabling headless integrations that collect answers via their own UI</li>
<li><code>tool_parameters</code> in OpenTelemetry tool_result events are now gated behind <code>OTEL_LOG_TOOL_DETAILS=1</code></li>
<li>Fixed <code>/compact</code> failing with "context exceeded" when the conversation has grown too large for the compact request itself to fit</li>
<li>Fixed <code>/plugin enable</code> and <code>/plugin disable</code> failing when a plugin's install location differs from where it's declared in settings</li>
<li>Fixed <code>--worktree</code> exiting with an error in non-git repositories before the <code>WorktreeCreate</code> hook could run</li>
<li>Fixed <code>deniedMcpServers</code> setting not blocking claude.ai MCP servers</li>
<li>Fixed <code>switch_display</code> in the computer-use tool returning "not available in this session" on multi-monitor setups</li>
<li>Fixed crash when <code>OTEL_LOGS_EXPORTER</code>, <code>OTEL_METRICS_EXPORTER</code>, or <code>OTEL_TRACES_EXPORTER</code> is set to <code>none</code></li>
<li>Fixed diff syntax highlighting not working in non-native builds</li>
<li>Fixed MCP step-up authorization failing when a refresh token exists — servers requesting elevated scopes via <code>403 insufficient_scope</code> now correctly trigger the re-authorization flow</li>
<li>Fixed memory leak in remote sessions when a streaming response is interrupted</li>
<li>Fixed persistent ECONNRESET errors during edge connection churn by using a fresh TCP connection on retry</li>
<li>Fixed prompts getting stuck in the queue after running certain slash commands, with up-arrow unable to retrieve them</li>
<li>Fixed Python Agent SDK: <code>type:'sdk'</code> MCP servers passed via <code>--mcp-config</code> are no longer dropped during startup</li>
<li>Fixed raw key sequences appearing in the prompt when running over SSH or in the VS Code integrated terminal</li>
<li>Fixed Remote Control session status staying stuck on "Requires Action" after a permission is resolved</li>
<li>Fixed shift+enter and meta+enter being intercepted by typeahead suggestions instead of inserting newlines</li>
<li>Fixed stale content bleeding through when scrolling up during streaming</li>
<li>Fixed terminal left in enhanced keyboard mode after exit in Ghostty, Kitty, WezTerm, and other terminals supporting the Kitty keyboard protocol — Ctrl+C and Ctrl+D now work correctly after quitting</li>
<li>Improved @-mention file autocomplete performance on large repositories</li>
<li>Improved PowerShell dangerous command detection</li>
<li>Improved scroll performance with large transcripts by replacing WASM yoga-layout with a pure TypeScript implementation</li>
<li>Reduced UI stutter when compaction triggers on large sessions</li>
</ul>
