## Summary
<h2>What's changed</h2>
<ul>
<li>Plugins in <code>.claude/skills</code> directories are now automatically loaded, no marketplace required</li>
<li>Added <code>claude plugin init &lt;name&gt;</code> to scaffold a new plugin in <code>.claude/skills</code></li>
<li>Added autocomplete for <code>/plugin</code> arguments: subcommands, installed plugin names, and plugins from known marketplaces</li>
<li><code>claude agents</code>: the <code>agent</code> field in <code>settings.json</code> is now honored for dispatched sessions, with <code>--agent &lt;name&gt;</code> to override it</li>
<li><code>EnterWorktree</code> can now switch between Claude-managed worktrees mid-session</li>
<li><code>tool_decision</code> telemetry events now include <code>tool_parameters</code> (bash commands, MCP/skill names) when <code>OTEL_LOG_TOOL_DETAILS=1</code></li>
<li>Worktrees managed by Claude are now left unlocked when the agent finishes, so <code>git worktree remove</code>/<code>prune</code> can clean them up</li>
<li>Fixed unprocessable images (zero-byte, corrupt) attached via paste, MCP, or dialog crashing the request instead of becoming a text placeholder</li>
<li>Fixed sandbox network permission prompts appearing in auto and bypass-permissions mode when using the desktop app, IDE extensions, or SDK</li>
<li>Fixed <code>claude agents</code> completed sessions not retiring when an idle subagent was still parked or had leaked a backgrounded shell</li>
<li>Fixed <code>claude agents</code> pressing Esc not cancelling a slow "opening…", leaving the list unresponsive</li>
<li>Fixed background agent worktrees under <code>.claude/worktrees/</code> being orphaned after the 30-day job retention sweep</li>
<li>Fixed background sessions re-attached after a sleep/wake not telling the model the correct date</li>
<li>Fixed copy-on-select in <code>claude agents</code> not reaching the system clipboard inside tmux with <code>set-clipboard on</code> (regression in 2.1.153)</li>
<li>Fixed <code>--resume</code> not reporting background subagents that were running when the previous Claude Code process exited</li>
<li>Fixed the <code>--resume</code> session picker leaving its contents on the terminal after exiting in fullscreen mode</li>
<li>Fixed <code>--worktree</code> and <code>--worktree --tmux</code> returning to the canonical repo root instead of the current linked worktree</li>
<li>Fixed the <code>/model</code> picker showing an incorrect "Newer version available" hint when the selected model is already the newest in its family; the pinned-model row now shows the model's description instead of its raw ID</li>
<li>Fixed literal markdown markers (backticks, asterisks) appearing in the in-progress message text in fullscreen mode</li>
<li>Fixed the terminal freezing after approving the managed-settings security dialog at startup</li>
<li>Fixed a rare duplicate line appearing in scrollback after the terminal UI redraws</li>
<li>Fixed right-click paste duplicating the clipboard in the VS Code, Cursor, and Windsurf integrated terminals</li>
<li>WSL: fixed image paste (<code>alt+v</code> keybinding), screenshot paste on Windows 11, and added support for dragging images from Windows Explorer</li>
<li>Improved performance of long and resumed conversations by eliminating redundant message-rendering recomputations</li>
<li><code>/terminal-setup</code> now disables GPU acceleration in VS Code/Cursor/Windsurf integrated terminals to prevent garbled-text rendering</li>
<li>The Feature of the Week credit-claim status now appears as a notification in the status area instead of a line above the prompt</li>
<li><code>claude agents</code>: slash-command autocomplete in the dispatch input now matches substrings</li>
<li>Removed the "bash commands will be sandboxed" startup banner — sandbox status still shows in <code>/status</code> and when a command is blocked</li>
<li>Removed the "/ide for …" startup hint toast</li>
<li>[IDE] Fixed clicking Stop while a background subagent is running not actually stopping it</li>
<li>[VSCode] Fixed the fast mode indicator not appearing on Opus 4.8</li>
<li>Pressing backspace right after a workflow trigger keyword now dismisses the workflow request (same as alt+w) instead of deleting a character</li>
<li>Added a "Workflow keyword trigger" setting in /config to stop the word "workflow" in a prompt from triggering a dynamic workflow</li>
</ul>
